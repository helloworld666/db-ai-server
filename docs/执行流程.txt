DB-AI-Server 执行流程 (v5.0 简化架构)

完整交互流程
==============

1. 用户输入阶段
---------------
用户: "查询所有用户"
    ↓
HTTP服务器接收请求 (/mcp/generate_sql)
    ↓
转发给SQL Agent处理

2. SQL Agent处理阶段 (简化版)
-----------------------------
# 步骤1: 获取数据库Schema
schema = schema_manager.get_all_table_names()
    ↓
# 步骤2: 构建提示词 (包含schema信息)
prompt = f"""
你是一个SQL生成助手...

【数据库结构】
表: users
  描述: 用户表
  字段:
    - id: int [PK] 用户ID
    - name: varchar 用户名
    ...

【规则】
1. 只能使用上述数据库结构中存在的表名和字段名
2. 禁止编造任何表名或字段名
...

【重要】
- 直接返回纯SQL语句，不要添加任何解释
- 不要返回JSON格式
- 不要返回Markdown代码块
"""
    ↓
# 步骤3: 调用LLM生成SQL
messages = [
    SystemMessage(content=prompt),
    HumanMessage(content="查询所有用户")
]
response = llm.invoke(messages)
    ↓
# 步骤4: 提取纯SQL
sql = extract_sql(response.content)
# 结果: "SELECT * FROM users"
    ↓
# 步骤5: 验证SQL
validation = sql_validator.validate(sql)

3. 返回结果阶段
--------------
Agent返回格式:
{
  "success": true,
  "sql": "SELECT * FROM users",
  "validation": {
    "is_valid": true,
    "errors": []
  }
}

HTTP响应:
{
  "success": true,
  "sql": "SELECT * FROM users"
}

架构对比
========

旧架构 (v4.x - 复杂)
--------------------
用户输入
    ↓
SQL Agent (ReAct循环)
    │
    ├── 1. LLM推理 → 选择工具
    │
    ├── 2. 工具调用 (get_schema)
    │   ↓
    ├── 3. ToolMessage (大JSON返回给LLM)
    │   ↓
    ├── 4. LLM被"训练" → 返回超大JSON
    │   ↓
    ├── 5. 多层解析失败
    │   ↓
    └── 6. 循环/错误

新架构 (v5.0 - 简化)
--------------------
用户输入
    ↓
SQL Agent (直接处理)
    │
    ├── 1. 获取Schema
    │   ↓
    ├── 2. 构建提示词
    │   ↓
    ├── 3. LLM生成SQL (纯文本)
    │   ↓
    ├── 4. 提取SQL
    │   ↓
    └── 5. 返回结果

关键改进
========

1. 移除ReAct循环
   - 不再有工具调用循环
   - 不再有ToolMessage大JSON问题
   - LLM只生成纯SQL，不生成工具调用JSON

2. 简化响应解析
   - 不需要解析复杂JSON
   - 不需要处理Markdown代码块
   - 不需要处理Python dict格式

3. 明确输出格式
   - 提示词明确要求"返回纯SQL"
   - 不返回JSON、不返回解释文字

错误处理
========

如果表不存在:
{
  "success": false,
  "error": "表 'xxx' 不存在",
  "sql": null
}

如果SQL验证失败:
{
  "success": false,
  "error": "SQL验证失败: [...]",
  "sql": "SELECT * FROM xxx"
}

LangChain v1.0 ReAct Agent 工具链实践
======================================

参考文档:
- https://langchain-doc.cn/v1/python/langchain/agents.html
- https://langchain-doc.cn/v1/python/langchain/agents/react.html

1. ReAct Agent 工具链调用
--------------------------
核心能力：LLM自主决定调用哪些工具、调用顺序

场景示例：
【查询】
  用户: "查询所有用户"
  工具链: generate_sql → execute_sql

【更新并返回】
  用户: "更新用户状态并返回更新后的数据"
  工具链: get_database_schema → generate_sql(UPDATE) → execute_sql → generate_sql(SELECT) → execute_sql

【添加并返回】
  用户: "添加新用户并返回添加的数据"
  工具链: get_database_schema → generate_sql(INSERT) → execute_sql → generate_sql(SELECT) → execute_sql

2. Agent创建标准
----------------
使用create_react_agent:
```python
from langchain.agents import create_react_agent, AgentExecutor
from langchain_core.tools import Tool

# 定义工具
tools = [
    Tool(name="generate_sql", func=..., description="生成SQL"),
    Tool(name="execute_sql", func=..., description="执行SQL"),
    Tool(name="get_database_schema", func=..., description="获取Schema"),
]

# 创建ReAct Agent
agent = create_react_agent(llm, tools, prompt)
agent_executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

# 执行（LLM自主决定工具链）
result = await agent_executor.ainvoke({"input": "更新用户并返回结果"})
```

3. 工具定义标准
---------------
使用Tool类:
```python
from langchain_core.tools import Tool

tools.append(Tool(
    name="generate_sql",
    func=generate_sql_function,
    description="根据自然语言生成SQL语句"
))
```
