# db-ai-server 代码规范与最佳实践

## 为什么需要代码规范？

**真实案例**：2026-04-10 出现的缩进错误导致"数据库操做描述不够具体"的错误：
- SQL检查通过，但由于缩进错误，错误提示代码始终执行
- 这是一个典型的低级错误，但影响严重

## 1. 自动化工具配置

### 1.1 代码格式化 - Black

**安装**：
```bash
pip install black
```

**使用**：
```bash
# 检查代码格式（不修改文件）
black --check --diff src/

# 自动格式化代码
black src/
```

**配置**（`pyproject.toml`）：
```toml
[tool.black]
line-length = 120
target-version = ['py310']
include = '\.pyi?$'
```

### 1.2 导入排序 - isort

**安装**：
```bash
pip install isort
```

**使用**：
```bash
# 检查导入顺序
isort --check-only --diff src/

# 自动排序导入
isort src/
```

### 1.3 统一编辑器配置 - EditorConfig

项目根目录已配置 `.editorconfig` 文件，确保团队成员使用一致的编辑器设置。

**支持的编辑器**：
- VS Code（自动识别）
- PyCharm（需要安装插件）
- Sublime Text（需要安装插件）
- Vim/Neovim（需要安装插件）

## 3. 代码风格指南

### 3.1 缩进

```python
# ✅ 正确：使用4个空格缩进
if condition:
    logger.info("Log message")
    return result

# ❌ 错误：使用制表符
if condition:
	logger.info("Log message")
	return result

# ❌ 错误：缩进不一致（本次问题的根源）
if condition:
    logger.info("Log message")

# 这段代码没有缩进，无论条件如何都会执行
error_msg = "错误信息"
return error_msg
```

### 3.2 行长度

```python
# ✅ 正确：每行不超过120字符
result = await mcp_client.call_tool(
    "execute_sql",
    {"sql": sql}
)

# ❌ 错误：行太长
result = await mcp_client.call_tool("execute_sql", {"sql": sql, "params": params, "timeout": 30})
```

### 3.3 字符串

```python
# ✅ 推荐：使用f-string
logger.info(f"SQL检查: {sql[:50]}...")

# ✅ 多行字符串使用三引号
error_msg = """数据库操做描述不够具体。
请提供更明确的描述，例如：
- '更新货架编号为***的货架'
- '更新用户名为***的用户为可用状态'
"""

# ❌ 避免字符串拼接（可读性差）
error_msg = "数据库操做描述不够具体。\n请提供更明确的描述，例如：\n- '更新货架编号为***的货架'"
```

## 4. 避免常见错误

### 4.1 缩进错误

**错误示例**：
```python
if re.search(pattern, text):
    logger.info("Pattern found")

# 这段代码没有缩进！
error_msg = "Error message"
return error_msg
```

**修复**：
```python
if re.search(pattern, text):
    logger.info("Pattern found")
    # 正确缩进
    error_msg = "Error message"
    return error_msg
```

**预防措施**：
- 使用编辑器的"显示空白字符"功能
- 运行 Black 自动格式化
- 使用 EditorConfig 统一缩进风格

### 4.2 复制粘贴错误

**错误场景**：
从不同文件/编辑器复制代码时，缩进可能丢失。

**预防措施**：
```bash
# 粘贴后立即格式化
black src/mcp_server.py
```

### 4.3 混用空格和制表符

**错误示例**：
```python
# 前两行用空格，第三行用制表符
if condition:
    logger.info("Log")
		return value  # 使用了制表符！
```

**预防措施**：
- 配置编辑器将制表符自动转换为空格
- 运行 Black 时会自动统一
